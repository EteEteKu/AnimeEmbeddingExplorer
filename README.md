## 📖 О проекте

**Anime Embedding Explorer** — это веб-приложение, разработанное в рамках портфолио, демонстрирующее применение методов **машинного обучения** и **вычислительной лингвистики** для задачи рекомендации аниме-тайтлов.

Основная идея заключается в том, чтобы по выбранному пользователем аниме находить другие произведения, наиболее близкие к нему по двум независимым "метрикам":

1. **Жанровая структура** (категориальное сходство).
2. **Сюжетно-смысловое содержание** (семантическая близость описаний).

Это позволяет выдавать релевантные рекомендации даже в случаях, когда жанры формально не пересекаются, но синопсисы имеют глубокое смысловое родство.

---

## 🧠 Методология

### 1. Метрика Жаккара (Жанровое сходство)

Для сравнения жанровых наборов используется **коэффициент Жаккара** — классическая мера схожести множеств:


J(A, B) = ∣A∪B∣/∣A∩B∣


где A и B — множества жанров двух аниме. Данная метрика обеспечивает строгое и интерпретируемое измерение пересечения категорий, инвариантное к размеру жанровых коллекций.

### 2. Семантическое сходство (Эмбеддинги синопсисов)

Для извлечения глубинного смысла из текстовых синопсисов применяется предварительно обученная **нейросетевая модель**, преобразующая текст в векторные представления (эмбеддинги). Мера косинусного расстояния между полученными векторами позволяет количественно оценить семантическую близость описаний, улавливая тематические и стилистические нюансы, которые не отражаются на уровне жанровых тегов.

### 3. Гибридная агрегация

Итоговый рейтинг рекомендаций вычисляется как взвешенная сумма нормализованных значений обеих метрик, что позволяет гибко настраивать баланс между структурным и содержательным подобием.

Приложение развернуто на платформе **Render**

При первом обращении к сайту время загрузки может составлять **до 30–45 секунд** — это нормально, так как происходит пробуждение контейнера и инициализация моделей. Последующие запросы будут обрабатываться с номинальной скоростью.

## 🧪 Запуск проекта локально

Для разработки и тестирования вам потребуется запустить как **бэкенд-сервер**, так и **фронтенд-клиент**.

### 1. Запуск бэкенда (FastAPI + Uvicorn)

```bash
# 1. Клонировать репозиторий
git clone https://github.com/your-username/anime-embedding-explorer.git
cd anime-embedding-explorer

# 2. Запустить сервер
uvicorn main:app --reload
```

После этого бэкенд будет доступен по адресу: `http://localhost:8000`

---

### 2. Настройка CORS (кросс-доменные запросы)

Для корректной работы фронтенда с локальным сервером необходимо разрешить CORS-запросы. В файле `main.py` добавьте или убедитесь, что присутствует следующая конфигурация:

```python
from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],  # разрешает запросы с любых источников (для разработки)
    allow_methods=["*"],  # разрешает все HTTP-методы
    allow_headers=["*"],  # разрешает все заголовки
)
```

> **💡 Важно:** В продакшене заменяйте `"*"` на конкретные домены (например, `["https://animeembeddingexplorer-1.onrender.com"]`) для безопасности. Для локальной разработки `"*"` — допустимый вариант.

---

### 3. Запуск фронтенда

Проект использует статический HTML-файл. Для локального запуска:

1. Откройте файл `index.html` в текстовом редакторе.
2. Найдите строку с базовым URL API (обычно в JavaScript-коде):
   ```javascript
   const API_BASE_URL = "https://animeembeddingexplorer-1.onrender.com";
   ```
3. Замените её на локальный адрес:
   ```javascript
   const API_BASE_URL = "http://127.0.0.1:8000";
   ```
   (или `http://localhost:8000`)
4. Сохраните файл и откройте его в браузере.
